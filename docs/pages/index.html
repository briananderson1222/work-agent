Platform

One shell for many agent systems.

Any runtime, one UI

Run managed agents, connected Claude and Codex sessions, and ACP-compatible external CLIs through one normalized orchestration model.

Plugins shape the product

Plugins can contribute layouts, agents, providers, tools, skills, knowledge namespaces, registries, settings, and branding.

Self-configuring agents

The built-in control surface lets agents create playbooks, manage skills, install integrations, schedule work, and coordinate other agents.

Local-first operations

Workspace data stays under the user's Stallion directory by default, while provider connections remain explicit and portable.

Knowledge and automation

Per-project vector knowledge, cron scheduling, notifications, voice, and MCP tools are available from the same workspace.

Observable by design

OpenTelemetry metrics and traces cover runtime behavior, provider decisions, costs, tools, scheduling, plugins, and UI actions.

Who it is for

From solo workspaces to internal agent platforms.

Individuals

Use one local workspace for preferred models, project knowledge, reusable playbooks, and runtime-specific agents.

Teams

Share project-scoped agents, knowledge, plugins, schedules, and verification workflows across repeatable work surfaces.

Organizations

White-label the shell, curate internal registries, connect enterprise auth, and keep data on infrastructure you control.

# Android Build Guide

Stallion AI ships as an Android APK built with [Tauri v2](https://tauri.app/) mobile support.

## Architecture

The Android build wraps the same web UI (Vite + React) in an Android WebView via Tauri. The native layer is minimal — Tauri generates the Android project at build time.

### Key directories

| Path | Purpose | Committed? |
|------|---------|------------|
| `src-desktop/` | Tauri config, Rust code, icons | Yes |
| `src-desktop/tauri.conf.json` | Tauri configuration | Yes |
| `src-desktop/src/lib.rs` | Rust entry point (desktop + mobile) | Yes |
| `src-desktop/gen/android/` | Generated Android project | No — generated by `tauri android init` |

### Important: `gen/android/` is ephemeral

The `gen/android/` directory is created by `npx tauri android init` and is NOT committed to the repository. It's generated fresh in CI on every build. Any native Android customizations must be applied as post-init patches in the CI workflow.

## CI Pipeline

The Android build runs via `.github/workflows/build-android.yml`.

### Pipeline steps

1. **Setup** — Node 20, Rust with Android targets, Java 17, Android SDK + NDK 27
2. **Build frontend** — `npm ci`, build SDK, build Connect, build UI
3. **Tauri Android Init** — `npx tauri android init` generates `gen/android/`
4. **Patch safe area** — Post-init script patches the generated Android theme to add `fitsSystemWindows`, preventing the WebView from rendering behind the status bar
5. **Build debug APK** — `npx tauri android build --debug --apk`
6. **Build release APK** — (tagged releases only) signed with `TAURI_SIGNING_PRIVATE_KEY`
7. **Upload** — APK uploaded as artifact (debug) or to GitHub Release (release)

### Triggers

- **Tag push** (`v*`) — builds debug + release APKs, uploads to GitHub Release
- **Manual dispatch** — builds debug APK only

## Status Bar / Safe Area

Android WebView does not reliably support CSS `env(safe-area-inset-*)` variables. On Android 15+ with edge-to-edge enforcement, the WebView content can render behind the status bar, making toolbar buttons unclickable.

### How we fix it

The CI pipeline patches the generated Android theme (`styles.xml`) after `tauri android init` to add `android:fitsSystemWindows="true"`. This tells Android to inset the WebView below the status bar at the native layer, so the web content never overlaps system UI.

The CSS layer also defines safe area variables as a belt-and-suspenders approach (primarily for iOS PWA support):

```css
:root {
  --safe-top: env(safe-area-inset-top, 0px);
  --safe-bottom: env(safe-area-inset-bottom, 0px);
}
```

## Local Development

### Prerequisites

- Android Studio (for SDK, NDK, emulator)
- Java 17+
- Rust with Android targets: `rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android`
- NDK 27: install via Android Studio SDK Manager or `sdkmanager "ndk;27.0.12077973"`

### Build locally

```bash
# Set NDK path
export NDK_HOME="$ANDROID_HOME/ndk/27.0.12077973"

# Build frontend
npm ci --legacy-peer-deps
cd packages/sdk && npm run build && cd ../..
cd packages/connect && npm run build && cd ../..
npm run build:ui

# Initialize Android project
npx tauri android init

# Build debug APK
npx tauri android build --debug --apk
```

The APK will be at `src-desktop/gen/android/app/build/outputs/apk/universal/debug/`.

### Run on emulator

```bash
# Start emulator (or connect a device)
adb devices

# Install and launch
adb install src-desktop/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
adb shell am start -n xyz.briananderson.stallion/.MainActivity
```

## Testing

Android-specific tests run via `.github/workflows/android-test.yml`:

- **Mobile Playwright** — Chromium with Pixel 7 viewport, tests in `tests/android/`
- **Emulator smoke test** — installs APK on Android 34 emulator, verifies app launches without crash

Run Playwright mobile tests locally:

```bash
npx playwright test --project=android
```

## Release Signing

Release APKs are signed using GitHub Secrets:
- `TAURI_SIGNING_PRIVATE_KEY` — the signing key
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — key password

These are configured in the repository's GitHub Settings > Secrets.
